# Feature → Advantage → Benefit (FAB) / 특징 → 장점 → 혜택 구조

- English name: Feature → Advantage → Benefit (FAB)
- Korean name: 특징 → 장점 → 혜택 구조
- Selection category: persuasion
- Use when: Use for product, service, or policy writing that must translate features into reader benefits.
- Avoid when: Avoid when the topic is not feature-driven.

## Writing Pattern

**Use a fixed three-step logic**

- Feature
  - Explain functions, specs, components, structure, and technology clearly.
  - Make it easy to see what’s included at a glance.
- Advantage
  - Emphasize our edge over competitors: usability, efficiency, performance.
  - Present evidence for what makes it better.
- Benefit
  - Explain customer value (time saved, cost reduced, satisfaction increased) in concrete terms.
  - Make the customer’s gain explicit.

**Each major section should include**

- Section heading
- Core paragraph purpose (1–2 sentences)
- Keep logic tight with connectors like “Thanks to this feature…” and “As a result, the customer…”.

## Example Writing Outline

1. **AI-Based Auto-Summarization**: An AI feature condenses uploaded documents into key sentences within three seconds. (Feature)
2. **Saving Time and Resources**: Compared to manual work, summarization time drops by over 85%, boosting team efficiency. (Advantage)
3. **Customers Focus on Strategy**: Freed from repetitive summarization, users can devote time to strategic planning. (Benefit)

## Writing Notes

- For Features, be specific and concise about technical traits (e.g., “cloud storage,” “high-resolution image engine”).
- For Advantages, make differentiation explicit—focus on competitive edge and user efficiency.
- For Benefits, write from the customer’s viewpoint and connect features/advantages to ultimate value.
- Ensure each step links logically and evokes both practical and emotional resonance—not just feature talk.
