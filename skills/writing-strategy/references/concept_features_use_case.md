# Concept → Features → Use Case / 컨셉 소개 → 기능 설명 → 사용 예시

- English name: Concept → Features → Use Case
- Korean name: 컨셉 소개 → 기능 설명 → 사용 예시
- Selection category: explanation
- Use when: Use when introducing a product, service, method, or idea through concept, features, and concrete use.
- Avoid when: Avoid when feature lists would overwhelm the reader.

## Writing Pattern

**Use this fixed three-step flow**

- Concept
  - Explain the core idea, philosophy, and starting context for the product or topic.
  - Answer “Why do we need this?” and “How is it different from the status quo?”
- Features
  - Describe the main features concretely and logically.
  - Don’t just list; emphasize how each feature connects to the concept and its strengths.
- Use Case
  - Show how the product or service is used in real scenarios.
  - Include user personas, environment, and context to strengthen clarity.

## Example Writing Outline

1. **AI-Assisted Article Workflow**: The workflow combines AI story design with outline recommendations for polished article drafting. (Concept)
2. **Three Core Features**: (1) Story structure suggestions, (2) automatic draft structuring, (3) brand-aligned tone guidance. (Features)
3. **A Planner’s Real Use Case**: Min-jung uploaded an existing report and finished an article in 10 minutes with the workflow; her proposal passed in the meeting. (Use Case)

## Writing Notes

- In the Concept step, clarify the product’s raison d’être and points of differentiation from existing solutions.
- In Features, communicate the value from a problem-solving perspective; group 3–4 core features and include associated benefits.
- In Use Case, be realistic and specific so the reader thinks, “I can use this.” Persona- or situation-centric storytelling works well.
- Maintain the arc of understanding → trust → empathy, so the reader can easily imagine “adopt → use → result.”
