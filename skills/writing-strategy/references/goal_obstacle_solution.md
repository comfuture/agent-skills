# Goal → Obstacle → Solution / 목표 → 장애물 → 극복 방안 (Goal → Obstacle → Solution)

- English name: Goal → Obstacle → Solution
- Korean name: 목표 → 장애물 → 극복 방안 (Goal → Obstacle → Solution)
- Selection category: planning
- Use when: Use when a goal is clear and the draft needs to show obstacles and how to overcome them.
- Avoid when: Avoid as a default solution frame when a more distinctive narrative fits. This is a generic structure; select it only after checking more distinctive options.

## Writing Pattern

**Use a fixed three-step flow**

- Goal
  - Define specific, measurable goals we aim to achieve.
  - Prefer schedules, outcomes, and KPIs over abstract visions.
  - Include why this goal matters.
- Obstacle
  - Analyze the main factors that block goal achievement.
  - Consider resource gaps, internal issues, external changes, and competitive pressure.
  - Don’t list everything—focus on the core obstacles.
- Solution
  - Present practical solutions and execution strategies to overcome obstacles.
  - Go beyond mitigation; turning obstacles into opportunities is even more persuasive.
  - Increase practicality by specifying actions, owners, and timing.

## Example Writing Outline

1. **Acquire 1,000 New Customers in 3 Months**: Based on a 3% conversion rate, we aim for 1,000 new signups within three months. (Goal)
2. **Main Obstacle — Low Landing-Page Conversion**: Traffic is sufficient, but a 65% landing-page bounce prevents conversion. (Obstacle)
3. **Solution — Content Redesign and A/B Tests**: We will overhaul content and reposition CTAs, targeting a conversion lift to 5%. (Solution)

## Writing Notes

- Express the Goal in specific numbers or states—set SMART goals, not “do our best.”
- Present Obstacles analytically, supported by data or real cases when possible.
- Solutions should include feasibility, ownership, and timing—and expected impact if possible.
- Keep the arc tight: “We’re heading for this goal → These obstacles exist → Here’s how we’ll respond.”
