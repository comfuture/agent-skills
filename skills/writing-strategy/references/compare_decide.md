# Compare → Decide / 비교 → 선택 구조

- English name: Compare → Decide
- Korean name: 비교 → 선택 구조
- Selection category: decision
- Use when: Use when readers must choose among options using explicit criteria.
- Avoid when: Avoid when there is only one serious option.

## Writing Pattern

**Structure the section flow using these criteria**

- Do not fix the number of sections; adjust to the number of alternatives and evaluation criteria.

**Follow this sequence**

- **Decision Context**: Explain what decision must be made
- **Comparison Criteria**: Present the basis for comparison
- **Option Analysis**: Compare each alternative’s pros/cons, figures, and cases
- **Recommended Choice**: Conclude what to select
- **Expected Impact or Plan**: Show outcomes of the choice or follow-up execution strategy

**Each major section should include**

- Section heading
- Core paragraph purpose (1–2 sentences)
- Base comparisons on objective indicators, reader priorities, and data-driven persuasion.
- When recommending, clearly state why the option is superior, using logical grounds rather than assertion.

## Example Writing Outline

1. **A Decision Is Required**: We are reviewing two solutions for a new customer management system.
2. **Comparison Criteria**: We evaluated cost, scalability, maintenance effort, and user satisfaction.
3. **Solution A Analysis**: Lower cost, but constraints around scalability and maintenance.
4. **Solution B Analysis**: Higher initial cost, but better flexibility and long-term operational stability.
5. **Recommendation and Rationale**: Considering long-term efficiency and stability, Solution B is the better fit.

## Writing Notes

- Ensure a natural flow from criteria → analysis → judgment, not a mere list.
- Prefer numbers, cases, and indicators over subjective language when comparing.
- Tie the recommendation to reader interests or objectives for stronger persuasion.
- Include next steps or an execution plan to answer “What should we do now?”
